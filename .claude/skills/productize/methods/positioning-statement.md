# Method — Positioning Statement (PRD 1.3)

*Re-authored from Dean Peters' Product-Manager-Skills (`positioning-statement`), CC BY-NC-SA 4.0 — attribution per Build Stance.*

Geoffrey Moore's frame: **For** [target] **who** [need], **the** [product] **is a** [category] **that** [key benefit]. **Unlike** [alternative], **it** [differentiator].

## Apply to PRD 1.3 (Product & Service Description)
- Product type + delivery model (from intake).
- **Core** features (the few that deliver the key benefit) vs. **secondary**.
- The **differentiator** — what makes it not-the-alternative. If you can't complete "Unlike … it …", the offering isn't differentiated yet — flag that in 1.9 rather than papering over it.

**Anti-pattern:** a feature list with no category or contrast. Positioning is about the gap you own.
