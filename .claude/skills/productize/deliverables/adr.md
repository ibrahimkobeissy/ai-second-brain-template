# Architecture Decision Record — deliverable spec

**Generates:** the key technical choices (language, framework, database, hosting) with rationale + trade-offs.
**Reads:** PRD 1.3 / 1.7 + `functional-spec`.
**Method:** one ADR per significant decision — context, options, decision, consequences. Be honest about what each choice trades away.
**Writes:** deliverable artifact per `templates/deliverable.md`.
