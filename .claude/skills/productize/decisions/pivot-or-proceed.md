# Pivot or Proceed — decision spec

**Evaluates:** alternative directions when signals are mixed — different segment, modified product, new pricing/model.
**Reads:** `feasibility`, `market-study`, `competitive-landscape`, `value-proposition`.
**Method:**
- Identify which dimension is dragging the case down (from the scorecard).
- Generate 2–3 concrete pivots that fix it; for each, what would have to be true and which analysis would re-test it.
- Prioritize against effort/evidence; recommend proceed-as-is, pivot-to-X, or shelve.
**Writes:** a decision artifact per `templates/decision.md`; `recommendation` = proceed / pivot-to-X / shelve, with the re-test plan.
