# Standup Tools

> **Area Context:** MOC and strategic hub for the Standup Tools area. **This is a `showcase` Area** — created to dogfood the productize toolkit end-to-end ([[productize-showcase]]); archivable once it has served its showcase + review purpose.

## 🎯 The Goal
Reduce the cost of status reporting for software teams — make "where are we?" cheap to produce and trustworthy to consume.

## 💡 The Idea & Scope
A durable space, not a single product: the broad problem of dev-team status/visibility. The first bet is [[standupzero]] (auto-drafted standups). *(Area challenge: this passes the area-vs-project test — the space outlives any one product; "StandupZero" is the project inside it.)*

**In Scope:** async standup / status tooling, the workflows around it, competitor intel, the productized bets.
**Out of Scope:** the actual app source code (built outside the vault per CLAUDE.md §7), real customer data, credentials.

## ⚙️ How (The Pipeline / Strategy)
Scout the space → curate signal into drafts → synthesize the bet → productize it (`productize-*`) → decide → spec. This Area is the staging ground for that flow.
